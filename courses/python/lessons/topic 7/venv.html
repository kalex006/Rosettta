7.5 Virtual Environments (venv)

In the professional world, you never install libraries globally. You use Virtual Environments to keep project dependencies isolated.

Why use venv?

Project A might need Django 3.0, while Project B needs Django 4.0. Installing globally would break one of them. venv solves this.

The Workflow

  1. Create: python -m venv myenv
  2. Activate (Windows): myenv\Scripts\activate
  3. Activate (Mac/Linux): source myenv/bin/activate
  4. Deactivate: Simply type deactivate

Once activated, any pip install you do stays inside that specific folder.
