8.3 Context Managers (with)

In the professional world, we rarely use f.close() manually. Instead, we use the with statement.

The "with" Keyword

The with statement creates a Context Manager. It automatically closes the file for you, even if an error occurs inside the block.

with open("test.txt", "r") as f:
    data = f.read()
    print(data)

# No need for f.close() here!

Why use this?

  • Safety: Prevents "Memory Leaks" (where a file stays open in the background).
  • Clean Code: It’s shorter and easier to read.
  • Standard: This is the "Pythonic" way to handle files.
