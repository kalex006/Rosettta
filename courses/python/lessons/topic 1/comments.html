1.5 Comments & Docstrings

Code is read more often than it is written. Comments help explain your logic to humans (and your future self).

Single Line Comments

Python uses the hash symbol # for single-line comments.

# This is a comment
print("Hello, World!") # This is also a comment

Multi-Line Comments (Docstrings)

Python does not have a specific multi-line comment syntax like C (`/* */`). Instead, we use triple quotes string literals.

"""
This is a docstring.
It is used to document functions, classes, and modules.
Python ignores string literals that are not assigned to a variable.
"""
print("Hello, World!")
