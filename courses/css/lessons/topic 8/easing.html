8.3 Easing & Cubic Bezier

The Calculus of Motion: Engineering momentum and weight through mathematical timing functions.

The Illusion of Life

In the physical world, objects require force to start moving and friction to stop. Constant, linear motion feels robotic and "cheap" to the human eye.

An architect uses Easing Functions to define how an animation's speed changes over its duration. By manipulating these curves, we can make the ROSETTA artifacts feel heavy like stone, or light like digital light.

1. Standard Timing Keywords

CSS provides several built-in keywords that represent pre-defined mathematical curves:

Keyword Physics Description Best Use Case
linear Constant speed. No acceleration. Color fades or background shifts.
ease-in Starts slow, ends fast. Objects leaving the screen.
ease-out Starts fast, ends slow. Objects entering the screen.
ease-in-out Slow start and slow end. Loops or internal state changes.

2. Deconstructing cubic-bezier()

For "Elite" motion, keywords are insufficient. The cubic-bezier(x1, y1, x2, y2) function allows you to draw your own acceleration curve on a 2D coordinate system.

  • x1, x2: Time coordinates (must be between 0 and 1).
  • y1, y2: Progress coordinates (can be greater than 1 to create "Bounce" effects).
⚡
The Rosetta "Heavy Stone" Curve:
For our artifacts, we use a custom curve that has a significant "lag" at the start to simulate weight:
cubic-bezier(0.7, 0, 0.3, 1).
CSS Architecture physics_tokens.css 
:root {
    /* Design Tokens for Motion */
    --ease-artifact: cubic-bezier(0.7, 0, 0.3, 1);
    --ease-out-back: cubic-bezier(0.34, 1.56, 0.64, 1); /* Bouncy entry */
}

.artifact-card:hover {
    transform: translateY(-10px);
    transition: transform 0.6s var(--ease-artifact);
}

3. Discrete Motion: steps()

Sometimes you don't want a smooth curve. The steps() function divides the animation into equal segments. This is the architectural standard for creating "Typewriter" effects or "Loading" indicators that feel like mechanical clocks.

CSS typewriter.css 
.terminal-text {
    /* Moves the cursor in 20 distinct jumps */
    animation: typing 4s steps(20) forwards;
}

4. Visualizing with DevTools

You cannot "guess" a bezier curve. All modern browsers (Chrome/Edge/Firefox) have a Bezier Visualizer built into the Inspector.

  1. Inspect an element with a transition.
  2. In the Styles pane, click the small curved icon next to the timing function.
  3. Drag the handles to feel the physics change in real-time.
⚠
The Golden Ratio of Duration: Large movements (like a modal opening) should take 300ms - 500ms. Small interactions (like a button hover) should take 100ms - 200ms. Anything faster than 100ms is imperceptible; anything slower than 500ms feels "sluggish."
