1.4 Inheritance & Initial

The Genetic Blueprint: Managing the propagation and termination of styles through the document hierarchy.

The Concept of Inheritance

In CSS architecture, Inheritance is the process by which certain properties are passed down from a parent element to its descendants. This allows for global consistency; for example, setting a font on the <body> ensures all text in the document shares that font without individual declarations.

However, not all properties are "hereditary." The browser's engine distinguishes between Inherited Properties and Non-Inherited Properties.

1. The Hereditary Divide

As a general rule, properties related to Typography and Color are inherited, while properties related to Box Geometry (Layout) are not.

  • Inherited: color, font-family, line-height, visibility, list-style.
  • Non-Inherited: border, margin, padding, width, background, position.
⚡
Architect's Logic: Imagine if border were inherited. If you put a border on a <div>, every paragraph and span inside it would also have a border. The layout would collapse into visual chaos. Non-inheritance is a safeguard for structural stability.

2. Explicit Control Keywords

CSS provides four powerful keywords that allow you to override the natural flow of inheritance for any property:

Keyword Action
inherit Forces the element to take the value from its parent, even for non-inherited properties.
initial Resets the property to the "factory default" of the CSS specification (often inline for display or black for color).
unset A hybrid: acts as inherit if the property is naturally inherited, otherwise acts as initial.
revert Rolls the style back to the User Agent Stylesheet (the browser's default style), ignoring your CSS.
CSS inheritance_overrides.css 
/* Force a button to use the same font as its parent container */
button {
    font-family: inherit;
}

/* Reset an element's margin to the CSS spec default (0) */
.artifact-reset {
    margin: initial;
}

/* Roll back a custom link to look like a blue browser link again */
.legacy-link {
    color: revert;
    text-decoration: revert;
}

3. The Nuclear Reset: all

The all property is a modern architectural tool that allows you to apply a keyword (like unset or revert) to every single CSS property of an element at once, except for unicode-bidi and direction.

CSS component_isolation.css 
/* Completely isolate a component from any global styles */
.widget-isolated {
    all: unset;
    /* Now start with a clean slate */
    display: block;
    background: #050505;
}
⚠
Performance Note: Overusing all: unset can be expensive for the rendering engine on low-powered devices, as it forces the browser to recalculate every property from scratch for that node. Use it only for critical component isolation.
