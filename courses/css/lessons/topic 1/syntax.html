1.1 The CSS Rule & Syntax

The Neural Blueprint: Deconstructing the atomic units of the Cascading Style Sheet.

The Rule-Set Anatomy

A CSS Rule-set is a declaration of intent. It tells the browser: "Find these specific elements, and apply these specific visual constraints to them."

Architecturally, a rule-set is composed of two primary parts: the Selector and the Declaration Block.

CSS anatomy_of_style.css 
/* The Selector */
.artifact-card {
    /* The Declaration Block */
    color: #d4af37;        /* Property: Value */
    border: 1px solid;    /* Semicolons are mandatory */
}

The Anatomy of a Declaration

Inside the curly braces {}, we find individual Declarations. Each declaration is a pair consisting of:

  • Property: The visual characteristic you wish to alter (e.g., font-family, background-color).
  • Value: The specific setting applied to that property (e.g., 'Cinzel', #050505).

Shorthand vs. Longhand

Efficiency is the hallmark of a Senior Architect. CSS provides Shorthand Properties that allow you to set multiple related properties in a single line.

Type Syntax Example
Longhand border-width: 1px; border-style: solid; border-color: gold;
Shorthand border: 1px solid gold;

While shorthands keep the style.css file small, be careful: shorthands often reset unspecified sub-properties to their initial values, which can cause bugs in complex cascades.

Case Sensitivity and Whitespace

CSS is largely case-insensitive. COLOR: GOLD; will function the same as color: gold;. However, by architectural convention, we strictly use lowercase.

The exceptions are Class Names and IDs, which can be case-sensitive depending on how the HTML is parsed. To avoid collision, ROSETTA uses kebab-case (e.g., .main-container) exclusively.

Architectural Commenting

In a large project, CSS becomes unmanageable without a structural map. CSS only supports block comments: /* comment */.

CSS documentation_style.css 
/* ==========================================
   TOPIC 01: CORE UI
   The primary theme for ROSETTA artifacts.
   ========================================== */

.artifact-title {
    font-family: 'Cinzel', serif;
}
