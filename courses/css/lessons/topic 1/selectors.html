1.2 Selectors & Combinators

Precision Targeting: Navigating the DOM hierarchy through relationship-based logic.

The Selection Engine

In CSS architecture, a Selector is a pattern used to identify the elements you want to style. While beginners rely solely on IDs and Classes, an Architect utilizes Combinators to describe the structural relationship between elements.

1. Simple Selectors

  • Type Selector: Targets tags directly (e.g., h1, p). High reach, low precision.
  • Class Selector: Targets elements with a specific class (e.g., .card). The workhorse of modern CSS.
  • ID Selector: Targets a unique ID (e.g., #main-nav). Extremely high precision, but limited reusability.
  • Universal Selector: (*) Targets every single element. Useful for global resets.

2. Combinatorial Logic

Combinators explain how two selectors relate to one another in the DOM tree. There are four primary combinators in the CSS specification:

Combinator Name Logic
A B Descendant Selects all B that are anywhere inside A.
A > B Child Selects all B that are direct children of A.
A + B Adjacent Sibling Selects B only if it is immediately after A.
A ~ B General Sibling Selects all B that follow A (must share parent).
CSS relationship_targeting.css 
/* Only style paragraphs that immediately follow a heading */
h2 + p {
    margin-top: 0;
    font-style: italic;
}

/* Style only the direct list items of a navigation menu */
nav > ul > li {
    display: inline-block;
}

3. Attribute Selectors

Attribute selectors allow you to target elements based on the presence or value of their HTML attributes. This is particularly powerful for styling forms or elements with data-* attributes in ROSETTA.

  • [attr]: Has the attribute.
  • [attr="val"]: Value is exactly "val".
  • [attr^="val"]: Value starts with "val".
  • [attr$="val"]: Value ends with "val".
  • [attr*="val"]: Value contains "val".
CSS attribute_selectors.css 
/* Style all external links (links starting with http) */
a[href^="http"] {
    color: #d4af37;
    text-decoration: underline dotted;
}

/* Target ROSETTA data-attributes */
[data-difficulty="advanced"] {
    border-left: 4px solid gold;
}

Architect's Performance Note: Browsers read CSS selectors from right to left. A selector like body div ul li a forces the browser to find every a tag on the page and then walk up the tree five levels to see if it matches. For high-performance sites, keep your selectors "shallow" (ideally 1-3 levels deep).
