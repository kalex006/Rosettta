9.1 Media Queries (Viewports)

Environmental Sensing: Orchestrating global layout shifts based on viewport dimensions and hardware capabilities.

The Adaptive Engine

Responsive Web Design (RWD) is the practice of building a single codebase that responds to the user's environment. The primary tool for this adaptation is the Media Query.

Think of a Media Query as a Conditional Logic Gate. It tells the browser: "If the screen is wider than X, or if the user is printing this page, apply these specific architectural overrides."

1. Syntactical Anatomy

A media query consists of a Media Type (optional) and one or more Media Features (expressions that resolve to true or false).

  • Media Types: all, print, screen.
  • Logical Operators: and, not, only, and the , (comma) which acts as an or.
CSS query_logic.css 
/* Only apply when the screen is between 600px and 1200px wide */
@media screen and (min-width: 600px) and (max-width: 1200px) {
    .artifact-grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

/* Apply when the user tries to print the documentation */
@media print {
    body { background: white; color: black; }
    .sidebar { display: none; }
}

2. Modern Range Syntax (2024+)

As a Senior Architect, you should adopt the modern **Range Syntax**. It is more readable and aligns with standard mathematical notation, making your stylesheets easier to maintain.

Modern CSS range_syntax.css 
/* Legacy Style */
@media (min-width: 768px) { ... }

/* Modern Architecture Style */
@media (width >= 768px) { 
    .artifact-container { padding: 40px; }
}

/* Range Logic */
@media (400px <= width <= 800px) {
    .artifact-title { font-size: 1.5rem; }
}

3. Breakpoint Strategy

A common architectural error is choosing breakpoints based on specific devices (e.g., "The iPhone Width"). Devices change every year. Instead, follow Content-Driven Breakpoints.

⚡
Architect's Protocol: Expand your browser window slowly. When the layout "breaks" (e.g., lines of text become too long to read, or the grid becomes too cramped), that is where your breakpoint belongs.

4. Standard Breakpoint Tokens

For the ROSETTA project, we recommend these semantic design tokens for consistency:

  • Mobile (Small): width < 600px
  • Tablet (Medium): 600px <= width < 1024px
  • Desktop (Large): width >= 1024px
  • Ultrawide: width >= 1600px

5. Beyond Width: Resolution & Orientation

Media queries can sense more than just size. You can target the orientation (landscape/portrait) or the resolution (for high-density "Retina" displays).

CSS hardware_sensing.css 
/* Targeting landscape tablets specifically */
@media (orientation: landscape) and (width < 1024px) {
    .sidebar { width: 100px; }
}

/* High-resolution assets for 2x displays */
@media (min-resolution: 2dppx) {
    .artifact-logo { background-image: url('logo@2x.png'); }
}
⚠
Architecture Note: Media queries only detect the Viewport (the whole window). They cannot detect how much space a specific component has. For that, we will learn about Container Queries in Lesson 9.3.
