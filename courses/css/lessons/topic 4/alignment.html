4.3 Alignment & Distribution

Orchestrating the Void: Mastering the mathematical distribution of space across dual axes.

The Physics of Positioning

In a Flexbox environment, alignment is not about fixed coordinates (x, y). Instead, it is about how the browser handles unused space. As an architect, you must decide what happens to the "void" when your items do not fill the entire container.

We manage this through two distinct properties: justify-content for the Main Axis and align-items for the Cross Axis.

1. Main Axis: justify-content

The justify-content property defines how space is distributed between and around flex items along the Main Axis.

Value Result
flex-start Items pack toward the start of the axis (Default).
flex-end Items pack toward the end of the axis.
center Items pack toward the center of the line.
space-between First item at the start, last item at the end, even gaps between.
space-around Even space around items (end-spaces are half the size of mid-spaces).
space-evenly All gaps, including edges, are mathematically equal.
CSS distribution_logic.css 
.navbar {
    display: flex;
    /* Logo on left, Navigation on right */
    justify-content: space-between; 
    align-items: center; /* Perfect vertical centering */
}

2. Cross Axis: align-items

The align-items property defines the default behavior for how flex items are laid out along the Cross Axis of the current line.

  • stretch (Default): Items expand to fill the container's cross-axis height.
  • center: Items are centered along the cross axis (The famous "Center a Div" method).
  • baseline: Items align based on the "sit-line" of their text content. Useful for mixing different font sizes.

3. Individual Overrides: align-self

Sometimes, a single artifact needs to break the rules. The align-self property is applied to the Flex Item, allowing it to override the container's align-items setting for just that specific node.

CSS rogue_alignment.css 
.container {
    display: flex;
    align-items: flex-start;
}

.item-highlight {
    /* This specific item ignores the container and sticks to the bottom */
    align-self: flex-end;
}

4. Multi-line Spacing: align-content

This property only takes effect when flex-wrap: wrap is enabled and there are multiple lines of items. It defines how the rows themselves are distributed within the container.

Architect's Trap: Beginners often confuse align-items with align-content. Remember: align-items is about how items sit within their single line. align-content is about how the lines themselves stack in the container.

5. The Modern Standard: gap

In the past, we used margin to create space between flex items, which often required :last-child { margin: 0; } hacks. The modern architect uses the gap property on the container.

The gap (or row-gap and column-gap) creates space only between items, never at the edges. It is much cleaner and mathematically precise.

CSS gap_architecture.css 
.artifact-grid {
    display: flex;
    flex-wrap: wrap;
    /* 20px between rows, 40px between columns */
    gap: 20px 40px; 
}
