3.2 Variable Fonts & Weights

The Fluid Typeface: Mastering the multi-axis mathematical spectrum of modern typography.

The Single-File Revolution

In the legacy era, every variation of a font (Thin, Regular, Bold, Black) required a separate HTTP request. For a high-end site like **ROSETTA**, this created a conflict between design richness and page performance.

Variable Fonts solve this by storing multiple variations within a single font file. Instead of fixed weights, we have a continuous Axis of possibilities. A variable font doesn't just have "Bold"; it has every value from 1 to 1000.

1. The Registered Axes

Most Variable Fonts (like Inter, which we use for our body text) support the five standard registered axes:

  • wght (Weight): Control the thickness of the strokes (e.g., 100 to 900).
  • wdth (Width): Control how condensed or expanded the letters are.
  • ital (Italic): A binary or range toggle for italicization.
  • slnt (Slant): Adjusts the angle of the letters (Oblique).
  • opsz (Optical Sizing): Automatically adjusts the font's design for different sizes (e.g., thicker stems for tiny text, thinner for headlines).
CSS variable_declaration.css 
@font-face {
    font-family: 'Inter Variable';
    /* Note the 'woff2-variations' format */
    src: url('../../fonts/Inter-Variable.woff2') format('woff2-variations');
    /* Define the range of supported weights */
    font-weight: 100 900;
    font-display: swap;
}

2. font-variation-settings

While you can use standard properties like font-weight: 542;, the most powerful way to manipulate a variable font is the font-variation-settings property. This allows you to tap into "Custom Axes" defined by the font designer.

CSS axis_control.css 
.artifact-heading {
    /* Precise control over Weight and Width */
    font-variation-settings: 'wght' 750, 'wdth' 110;
    
    /* Smooth transition between weights */
    transition: font-variation-settings 0.5s ease;
}

.artifact-heading:hover {
    font-variation-settings: 'wght' 900, 'wdth' 125;
}

3. Optical Sizing (opsz)

This is the "Senior Architect" secret to premium UI. In the physical world of printing, tiny letters were carved differently than huge headlines to ensure legibility.

With font-optical-sizing: auto;, the browser detects the font-size and automatically adjusts the variable font's internal geometry. It makes small text more readable and large text more elegant.

4. Performance: The Font-Weight Tax

A single Variable Font file is usually larger than a single static font file (e.g., 200kb vs 30kb). However, it is significantly smaller than the **sum** of 4 or 5 static files.

⚡
Architect's Recommendation: Use Variable Fonts for your primary UI font (Inter) to allow for subtle weight changes in hover states and animations. Use static fonts only for highly decorative, single-weight titles (like Cinzel).
⚠
Legacy Fallback: Always provide a standard font-weight fallback for browsers that do not support variable axes. The browser will ignore font-variation-settings if it doesn't understand them.
