3.3 RGB, HEX, HSL & OKLCH

Chromatographic Engineering: Mastering perceptual uniformity and the expanded P3 color gamut.

The Evolution of Color Models

Color in CSS has historically been defined by how hardware works (Red, Green, Blue LEDs). However, a Senior Architect must define color by how the human eye works.

Standard color spaces like sRGB (used by HEX and RGB) can only display about 35% of the colors the human eye can see. Modern displays support Display P3, and modern CSS gives us the mathematical tools to unlock these "High Definition" colors.

1. The Legacy: HEX and RGB

HEX (#d4af37) and RGB (rgb(212, 175, 55)) are essentially the same. They are coordinate-based systems that tell the monitor how much light to blast from each LED.

  • Pros: Universal support; easy to copy from design tools.
  • Cons: Impossible for a human to "read." You cannot look at a HEX code and know how to make it 10% darker just by changing the numbers in your head.

2. The Perceptual Flaw of HSL

HSL (Hue, Saturation, Lightness) was a major improvement because it is intuitive. However, HSL is mathematically "dishonest."

⚠
The HSL Glitch: In HSL, a "Lightness" of 50% for Yellow looks much brighter to the human eye than a "Lightness" of 50% for Blue. This makes creating consistent accessible UI (contrast ratios) very difficult.

3. The Future: OKLCH

OKLCH is the new architectural standard. It is a Perceptually Uniform color space. If you change the Hue, the perceived brightness stays exactly the same.

  • L (Lightness): 0% (Black) to 100% (White).
  • C (Chroma): The "amount" of color. Unlike saturation, this has no theoretical cap.
  • H (Hue): The angle on the color wheel (0-360).
CSS modern_colors.css 
/* Defining our Ancient Gold in OKLCH */
:root {
    --gold-hd: oklch(75% 0.15 85); 
}

.artifact-card {
    /* Making it 10% darker is mathematically simple */
    background: oklch(65% 0.15 85);
    
    /* Accessing colors that don't exist in sRGB */
    color: oklch(70% 0.5 20); /* A neon red impossible in HEX */
}

4. Color Gamuts: sRGB vs. P3

By using the color() function or oklch(), we can access the P3 Gamut. P3 contains 25% more colors than sRGB, specifically in the vibrant greens and deep reds.

For the ROSETTA project, using P3 colors on high-end devices (MacBooks, iPhones, OLED screens) makes the "Ancient Gold" appear to glow with a metallic vibrance that is impossible with standard HEX codes.

5. Color Mixing

The modern color-mix() function allows you to blend colors directly in the browser. This is essential for creating "Tint" and "Shade" systems without manually picking new HEX codes.

CSS dynamic_mixing.css 
.artifact-overlay {
    /* Mix 20% gold into a black background */
    background: color-mix(in oklch, var(--gold-hd), black 80%);
}
⚡
Architect's Recommendation: Use OKLCH for your design system variables. It allows you to generate accessible color palettes (ensuring 4.5:1 contrast) programmatically by simply adjusting the L (Lightness) value.
