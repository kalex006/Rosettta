3.1 Sectioning Roots: Body & Main

The Canvas and the Core: Establishing the primary boundaries of the visual document.

The Physical Canvas: <body>

The <body> element represents the content of the document. Structurally, there can be only one body element in a document.

It is a Sectioning Root. This means it establishes its own independent outline. Headings inside the body start the "Table of Contents" for the page.

The Primary Landmark: <main>

Inside the body, we have headers, footers, sidebars, and ads. But the reason the user visited the page is contained within the Main Content Area.

The <main> element wraps this specific content. It is architecturally critical for Assistive Technology (Screen Readers). When a blind user presses a "Skip to Content" key, the browser looks for this specific element.

HTML document_anatomy.html 
<body>
    <header>Global Navigation (Not unique)</header>

    <!-- The core reason for the page's existence -->
    <main>
        <h1>The Article Title</h1>
        <p>The content...</p>
    </main>

    <footer>Copyright Info (Not unique)</footer>
</body>

The "Uniqueness" Constraint

A document must not have more than one visible <main> element.

You can technically have multiple main tags in the DOM (for example, if you are pre-loading the next page's view in a Single Page Application), but all except one must be hidden using the hidden attribute.

HTML spa_architecture.html 
<main id="view-home" hidden>...</main>
<main id="view-dashboard">Active Content</main>
<main id="view-settings" hidden>...</main>

Why not just use <div id="main">?

Before HTML5, we used <div id="main">. While this looks the same visually, it lacks semantic weight.

The <main> tag automatically maps to the ARIA role="main". This exposes it to the Accessibility Tree without you writing extra code. Using a generic `div` requires you to manually add `role="main"`, which is redundant and error-prone.
