3.4 The Document Outline Algorithm

The Failed Dream: Understanding why the H1-H6 hierarchy still rules the web.

The HTML5 Theoretical Outline

When HTML5 was first drafted, it proposed a revolutionary idea: The Outline Algorithm. The concept was that every time you opened a new <section> or <article>, the heading level would reset logically.

In theory, this meant you could use <h1> everywhere, and the browser would automatically calculate that an <h1> inside a nested section should effectively act like an <h2>.

HTML theoretical_spec.html 
<!-- ❌ THEORETICAL (DO NOT USE) -->
<body>
    <h1>Page Title (Level 1)</h1>
    <section>
        <h1>Section Title (Theoretically Level 2)</h1>
        <article>
             <h1>Article Title (Theoretically Level 3)</h1>
        </article>
    </section>
</body>

The Hard Reality

Browsers and Screen Readers never implemented this algorithm.

To an assistive technology user, the code above looks like three top-level headings of equal importance. This creates a confusing navigation experience where the "Article Title" sounds just as important as the "Page Title."

As Architects, we must enforce a strict Hard Hierarchy using the numbered tags: <h1> through <h6>.

The Rules of Rank

  • The H1: There should generally be only one <h1> per page. It represents the main topic of the document.
  • No Skipping: Do not jump from <h2> to <h4> just because you like the font size of the h4 better. Use CSS to change the size; use HTML to define the rank.
  • Depth Limit: While valid, if you find yourself using <h6>, your content structure is likely too complex and should be refactored into separate pages.
HTML practical_structure.html 
<!-- ✅ PRACTICAL (DO THIS) -->
<body>
    <h1>Project ROSETTA</h1>
    
    <section>
        <h2>Course Syllabus</h2>
        
        <article>
            <h3>Module 1: Architecture</h3>
            <p>Content...</p>
        </article>
    </section>
</body>

The Hgroup Resurrection

Sometimes a heading has a subtitle or a tagline. Historically, developers wrapped both in a `div`. Recently, the standard brought back the <hgroup> element.

It allows you to group a heading with a paragraph that acts as a subtitle/tagline, ensuring the subtitle doesn't clutter the document outline navigation.

HTML hgroup_syntax.html 
<hgroup>
    <h1>Project ROSETTA</h1>
    <p>A Zero-Backend Architecture</p>
</hgroup>
⚡
Architect's Tip: Most accessibility audit tools (like Lighthouse or Axe) will flag "Skipped Heading Levels" as an error. Always maintain the strict integer sequence of levels.
