3.3 Article vs Section vs Div

The Hierarchy of Meaning: A decision matrix for structural semantics.

The Decision Matrix

When an Architect builds a layout, they must ask specific questions to determine which container to use. It is not about how it looks (CSS handles that); it is about what it represents to the machine.

1. The Independent Entity: <article>

The <article> element represents a self-contained composition.

The Litmus Test: "Could this block of content be distributed independently via an RSS feed?"

If you ripped this element out of the page and pasted it onto another website or sent it in an email, would it still make sense? If yes, it is an Article.

  • A blog post.
  • A user comment (yes, comments are articles).
  • A weather widget.

2. The Thematic Grouping: <section>

The <section> element represents a generic section of a document or application. A section, in this context, is a thematic grouping of content.

The Litmus Test: "Does this grouping naturally have a heading?"

According to the W3C spec, a section typically has a title. If you are grouping elements just to put a border around them, it is probably not a section.

3. The Generic Container: <div>

The <div> has no semantic meaning. It conveys nothing to the accessibility tree.

Usage: Use it strictly for layout purposes. If you need a wrapper for a CSS Grid, a Flexbox container, or a styling hook for a background color, use a Div.

HTML nesting_logic.html 
<!-- A News Feed -->
<section class="news-feed">
    <h2>Latest Tech News</h2> <!-- Section has a heading -->

    <!-- A Self-Contained Story -->
    <article class="news-item">
        <h3>New Chip Architecture Released</h3>
        <p>Intel has announced...</p>
        
        <!-- Generic wrapper for layout styling -->
        <div class="share-buttons">
            <button>Share</button>
        </div>
    </article>

</section>

Recursive Nesting

It is perfectly valid to nest these elements recursively.

  • An <article> can contain multiple <section> elements (e.g., Introduction, Methodology, Conclusion chapters).
  • A <section> can contain multiple <article> elements (e.g., a "Comments" section containing individual comment articles).
