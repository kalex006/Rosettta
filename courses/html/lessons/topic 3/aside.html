3.5 Aside & Complementary Content

Tangential Relevance: Managing content that exists outside the direct flow.

The Definition of "Tangential"

The <aside> element represents a portion of a document whose content is only indirectly related to the document's main content.

The Litmus Test: If you remove this element, does the main article still make perfect sense?

  • Yes: It is likely an <aside> (e.g., an advertisement, a "related posts" widget, a pull quote).
  • No: If removing it breaks the understanding of the text, it should be part of the main flow (e.g., a critical diagram or definition).

Contextual Scoping

Like headers and footers, the meaning of an Aside changes based on where it lives in the DOM hierarchy.

HTML scoping_logic.html 
<body>
    <!-- 1. GLOBAL ASIDE (Site-level) -->
    <aside class="sidebar">
        <h3>About the Author</h3>
        <h3>Advertising</h3>
    </aside>

    <article>
        <h1>The History of CSS</h1>
        <p>CSS was proposed by Håkon Wium Lie...</p>

        <!-- 2. LOCAL ASIDE (Article-level) -->
        <aside class="glossary-term">
            <h4>Did you know?</h4>
            <p>Before CSS, tags like <font> were used for styling.</p>
        </aside>
    </article>
</body>

The Accessibility Mapping

Browsers map the <aside> element to the Complementary role in the accessibility API.

This allows screen reader users to jump directly to it if they are interested in extra info, or skip it entirely if they just want to read the main story. This is why you should never put critical information (like the "Buy Now" button) inside an aside.

Pull Quotes

A common aesthetic pattern is the "Pull Quote"—a sentence from the text repeated in large font for visual interest.

If the quote is simply decorative repetition, use <aside> combined with aria-hidden="true" so screen readers do not read the same sentence twice.

HTML pull_quote.html 
<aside class="pull-quote" aria-hidden="true">
    "The aside element is about relationship, not position."
</aside>
⚡
Architect's Tip: Do not use <aside> for parentheses content inside a paragraph `(like this)`. For that, use the <small> tag or simple text. The Aside element is a block-level container, not an inline modifier.
