1.4 Content Categories

Beyond Block and Inline: Understanding the strict content models that govern validity.

The Paradigm Shift

Historically, HTML elements were categorized as either Block (takes full width) or Inline (sits within text). In modern architecture, these terms define CSS Display modes, not HTML validity.

HTML5 introduced a more nuanced Content Model to define exactly which elements can dwell inside others.

1. Flow Content

This is the broadest category. It includes almost every element that can be used in the body of a document. If an element accepts "Flow Content," it is structurally very permissive.

Examples: <div>, <p>, <ul>, <table>.

2. Phrasing Content

Phrasing content is the text of the document and the elements that mark up that text at the intra-paragraph level. This roughly correlates to the old "inline" concept.

Examples: <span>, <strong>, <em>, <img>.

⚠
The Golden Rule: Phrasing content cannot contain Flow content. You cannot put a <div> inside a <span> without violating semantic rules (even if CSS makes it look correct).

The Paragraph Predicament

The <p> element is unique. It can only contain Phrasing Content. If you attempt to place a Flow Content element (like a <div>) inside a paragraph, the browser's parser will aggressively intervene.

HTML parser_behavior.html 
<!-- WHAT YOU WRITE -->
<p>
    Hello World
    <div>I am an illegal block</div>
</p>

<!-- WHAT THE BROWSER RENDERS (DOM) -->
<p>Hello World</p> <!-- Auto-closed here -->
<div>I am an illegal block</div>
<p></p> <!-- Stray closing tag creates an empty p -->

3. Transparent Content Models

Some elements, most notably the Anchor <a>, have a Transparent model. They adopt the content model of their parent.

This means you can wrap an entire card (divs, images, headings) inside a single link tag in HTML5—a practice that was illegal in HTML4.

4. Interactive Content

Elements specifically designed for user interaction (buttons, links, inputs). The architectural constraint here is strict: Interactive content cannot be nested.

You generally cannot place a <button> inside an <a> tag, as the browser cannot decide which interaction takes precedence.
