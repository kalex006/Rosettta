1.1 The Web Standard & Doctypes

The Protocol of the Web: From SGML roots to the Living Standard.

The Living Standard

HTML is no longer a versioned language like Python or C++. Since 2019, the WHATWG (Web Hypertext Application Technology Working Group) has declared HTML a "Living Standard."

This means there will never be an "HTML6." Instead, the browser vendors (Chrome, Firefox, Safari, Edge) continuously merge new features into the engine. As architects, we do not write for a version; we write for the platform.

The Doctype Declaration

The very first line of any professional HTML document is the DOCTYPE. It is crucial to understand that this is not an HTML tag. It is a preamble instructions for the browser's parser.

Historically, HTML was based on SGML (Standard Generalized Markup Language), which required a reference to a DTD (Document Type Definition). Modern HTML is its own thing, but the Doctype remains for a specific architectural reason: Rendering Modes.

HTML index.html 
<!-- 
  The Modern Declaration.
  Case-insensitive, but uppercase is conventional.
  Triggers "Standards Mode" in all browsers.
-->
<!DOCTYPE html>

<html lang="en">
...
</html>

Quirks vs. Standards Mode

When a browser loads a page, it decides which layout engine to use based on the presence of the Doctype.

  • Standards Mode: Triggered by `<!DOCTYPE html>`. The browser renders the page according to modern CSS and HTML specifications.
  • Quirks Mode: Triggered if the Doctype is missing or malformed. The browser mimics the behavior of Navigator 4 and IE5 to support ancient websites.

The Root Element

Immediately following the declaration is the root element: <html>. All other elements are descendants of this node. It is best practice to always include the lang attribute (e.g., lang="en"). This is not just for semantics; it is vital for accessibility tools (screen readers) and search engine logic.
