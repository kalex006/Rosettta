9.1 Details & Summary Disclosure

Native State Management: Implementing the disclosure pattern for hierarchical information density.

The Disclosure Widget

The <details> and <summary> elements provide a native way to create "Accordions" or "Collapsible" regions. Historically, this required jQuery or complex Vanilla JS state-toggling.

In the ROSETTA architecture, we use these to manage Information Density—hiding advanced technical specifications until the user explicitly requests them, thereby keeping the primary "Artifact" interface clean and focused.

1. Structural Anatomy

  • <details>: The wrapper that manages the open/closed state.
  • <summary>: The visible "Label" or "Trigger." Clicking this toggles the visibility of the rest of the content inside the details tag.
HTML disclosure_pattern.html 
<details>
    <summary>View Encrypted Metadata</summary>
    <div class="artifact-data">
        <p>Origin: Northern Sector</p>
        <p>Timestamp: 2026-02-01</p>
    </div>
</details>

2. The 'open' Attribute

The state of the widget is reflected in the DOM via the open boolean attribute.

Architectural Benefit: Because the state is in the HTML, you can use CSS selectors to style the element based on whether it is expanded. This is the foundation of "Pure CSS" interactive UI.

CSS state_styling.css 
/* Style changes when the artifact is 'unlocked' (opened) */
details[open] {
    border-color: #d4af37; /* Ancient Gold */
    background: rgba(212, 175, 55, 0.05);
}

details[open] summary {
    margin-bottom: 20px;
    font-weight: 700;
}

3. Styling the Marker

By default, browsers show a small triangle (arrow) next to the summary. As an Architect, you can customize or remove this to fit the high-end aesthetic.

  • Use the ::marker pseudo-element for standard styling.
  • Use summary::-webkit-details-marker for older WebKit browsers.
  • To hide it entirely, set display: none or list-style: none on the summary.

4. Accessibility & Searchability

Unlike JavaScript accordions that often use display: none, content inside a <details> tag is "Find-in-Page" friendly.

⚡
Native Search: If a user uses Ctrl+F to search for a word that is hidden inside a closed details widget, most modern browsers will automatically open the widget and scroll to the text. This is a massive SEO and UX advantage over custom JS solutions.

The Exclusive Accordion (New Feature)

A recent addition to the HTML spec allows you to give multiple <details> elements the same name attribute. This creates an "Exclusive Accordion" where opening one automatically closes the others in the group.

HTML exclusive_group.html 
<details name="rosetta-group">
    <summary>Section I</summary> ...
</details>

<details name="rosetta-group">
    <summary>Section II</summary> ...
</details>
⚠
Architect's Limit: Do not use <details> for mission-critical information that must be seen immediately. It is for progressive disclosure—secondary information that supports the primary narrative.
