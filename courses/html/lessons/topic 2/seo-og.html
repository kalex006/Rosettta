2.3 SEO Intelligence & Open Graph

The Social Logic: Controlling link unfurling and search engine snippets.

The Search Snippet

When Google indexes a page, it needs a summary to display in the SERP (Search Engine Results Page). This is controlled by the description meta tag.

While Google has confirmed that the description is not a direct ranking factor, it is a massive CTR (Click-Through Rate) factor. A well-engineered description acts as ad copy, enticing users to choose your link over a competitor's.

HTML seo_basic.html 
<meta name="description" content="Master advanced HTML5 architecture with ROSETTA. A zero-backend, dark-themed documentation platform for serious engineers.">
<meta name="keywords" content="HTML, DOM, Architecture"> <!-- Mostly ignored by modern Google -->
<meta name="author" content="Kalex">

The Open Graph Protocol (OG)

In the early web, sharing a link just pasted the blue underlined text. Today, platforms like Facebook, Discord, and LinkedIn "unfurl" links into rich cards with images and titles.

This is powered by the Open Graph Protocol, originally created by Facebook. It effectively turns your web page into a rich object within the social graph. Without these tags, the social crawler will guess (poorly) which image and text to display.

HTML social_graph.html 
<!-- Essential OG Tags -->
<meta property="og:type" content="website">
<meta property="og:url" content="https://rosetta.com/courses/html">
<meta property="og:title" content="Advanced HTML Architecture">
<meta property="og:description" content="A deep dive into the DOM and Rendering Engine.">
<meta property="og:image" content="https://rosetta.com/assets/og-banner.jpg">

<!-- Twitter/X Specifics -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:creator" content="@kalex_dev">

The "Unfurling" Mechanics

When a user pastes a link into Discord:

  1. The Discord Bot (User-Agent: `Discordbot`) makes a GET request to your URL.
  2. It downloads just the HTML (it usually does not execute JavaScript).
  3. It parses the <head> looking for `og:*` properties.
  4. It constructs the card UI based on the metadata found.

The 1200x630 Standard

For the `og:image`, consistency is key. The industry standard resolution is 1200px by 630px (an aspect ratio of 1.91:1).

If your image is too small, platforms will render a tiny thumbnail instead of the "Large Image" card, significantly reducing engagement.
