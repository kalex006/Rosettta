7.1 SVG Namespace & Structure

Mathematical Imagery: Mastering the XML-based vector coordinate system.

The Vector Paradigm

Unlike Raster images (JPEG/PNG) which are grids of colored pixels, SVG (Scalable Vector Graphics) is a text-based description of geometric shapes.

Because it is defined by mathematics rather than a pixel grid, an SVG artifact can be scaled to the size of a skyscraper or shrunk to the size of a grain of sand without losing a single degree of sharpness. In the ROSETTA architecture, SVGs are the preferred medium for icons, diagrams, and complex decorative flourishes.

The Namespace: xmlns

SVG is a dialect of XML. Historically, browsers needed to be told explicitly which "dictionary" to use to translate the tags. This is done via the xmlns (XML Namespace) attribute.

Architect's Note: When you paste an SVG inline inside an HTML5 document, the xmlns attribute is optional because the HTML5 parser is smart enough to recognize the <svg> tag. However, if the SVG is a standalone file (e.g., icon.svg), the namespace is mandatory.
XML/SVG standalone_artifact.svg 
<?xml version="1.0" encoding="UTF-8"?>
<!-- The namespace is required for standalone files -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="40" fill="#d4af37" />
</svg>

The Coordinate System

The SVG world starts at the Top-Left Corner (0,0).

  • X-axis: Increases to the right.
  • Y-axis: Increases downward (this is the opposite of many mathematical Cartesian systems).

Mastering the ViewBox

The viewBox is the most critical attribute in SVG architecture. It defines the "internal aspect ratio" and the "visible window" of the coordinate system.

The value viewBox="min-x min-y width height" acts like a camera lens. It tells the browser: "Map this internal 100x100 unit coordinate system to whatever size the container actually is."

HTML/SVG viewbox_mapping.html 
<!-- 
  Internal units: 0 to 100. 
  External CSS size: 500px.
  The browser calculates that 1 SVG unit = 5px.
-->
<svg viewBox="0 0 100 100" style="width: 500px;">
    <rect x="10" y="10" width="80" height="80" fill="currentColor" />
</svg>

Core Structural Elements

Every complex SVG is built from these basic primitives:

  • <g>: The Group tag. Used to apply styles (like colors or transforms) to multiple elements at once.
  • <defs>: The Definitions area. Used to store reusable elements (like gradients or patterns) that aren't rendered immediately.
  • <use>: Clones an element from <defs> and places it elsewhere in the document.

The "DOM-Nature" of SVG

Unlike a <canvas> (which is just a "bucket of pixels"), every element inside an <svg> is a real DOM node. This means you can select a <circle> using JavaScript (document.querySelector) and add event listeners to it, making SVGs perfect for interactive maps and charts.
