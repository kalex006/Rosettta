7.2 Inline SVG vs. Object vs. Img

Deployment Strategies: Evaluating the three pillars of vector integration.

The Integration Dilemma

In the ROSETTA architecture, SVGs are everywhere—from the language logos on the landing page to the small icons in the sidebar. However, how you "inject" these vectors into the document significantly impacts both performance and flexibility.

There is no "perfect" method; there is only the correct method for the specific context.

1. The Image Tag (<img>)

The simplest method. You treat the SVG like a standard JPEG or PNG.

  • Pros: Excellent caching (browser saves the file), keeps HTML clean, supports alt text.
  • Cons: Zero interaction. You cannot change the color of the SVG via CSS, and you cannot select internal elements with JavaScript.
HTML static_vector.html 
<!-- Use for static icons that don't need hover effects or color changes -->
<img src="assets/icons/rust-logo.svg" alt="Rust Programming Language">

2. The Inline Method (<svg>)

Pasting the raw <svg> code directly into your HTML document.

  • Pros: Total control. You can style every path with CSS and animate them with JS. It also reduces HTTP requests (no extra file download).
  • Cons: Bloats the HTML file, prevents caching, and can make the source code difficult to read.
HTML/CSS interactive_vector.html 
<!-- Use for UI elements with complex hover states -->
<svg class="artifact-icon" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="40" />
</svg>

<style>
    .artifact-icon:hover circle {
        fill: #d4af37; /* The "Ancient Gold" shift */
        transition: fill 0.5s ease;
    }
</style>

3. The External Object (<object>)

The <object> element embeds an SVG as its own document, similar to an iframe, but specifically for media.

  • Pros: Maintains caching while allowing some level of external CSS/JS interaction (via the contentDocument API).
  • Cons: More complex to code; requires scripts to "reach into" the object to manipulate it.
HTML object_integration.html 
<object type="image/svg+xml" data="map.svg" class="interactive-map">
    <!-- Fallback content -->
    <img src="map.png" alt="Static Map">
</object>

Architectural Comparison

Feature <img> Inline <svg> <object>
Caching Yes (Excellent) No Yes
CSS Styling None Full (External CSS) Limited (Internal only)
JS Interaction None Full (DOM access) Moderate
Accessibility via alt via <title> via aria-label
⚡
The "currentColor" Trick: If you use Inline SVGs, set the fill or stroke of your paths to currentColor in the SVG code. This allows the icon to automatically inherit the text color of its parent container, making theme management (Light/Dark) effortless.
