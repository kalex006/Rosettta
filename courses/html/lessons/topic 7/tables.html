7.3 Table Anatomy

Data Serialization: Implementing the tripartite structural model for complex tabular data.

The Semantic Table

In modern frontend architecture, tables have a singular purpose: The display of multi-dimensional, tabular data.

Historically, tables were abused to create page layouts. Today, that practice is strictly prohibited. A table is a data-structure visualization. To ensure that browsers and assistive technologies can navigate these structures, we must use the explicit semantic grouping elements: <thead>, <tbody>, and <tfoot>.

1. The Table Head (<thead>)

The <thead> defines the labels for the columns. It must contain rows (<tr>) which in turn contain header cells (<th>).

Architectural Benefit: When printing long tables, modern browsers will repeat the <thead> at the top of every new physical page automatically.

2. The Table Body (<tbody>)

The <tbody> encapsulates the primary data of the table. A single table can actually contain multiple <tbody> elements to represent different sections or "chapters" of data within the same header structure.

3. The Table Foot (<tfoot>)

The <tfoot> contains summary information, such as totals, averages, or concluding signatures.

⚠
Legacy Note: In older versions of HTML, the <tfoot> had to be placed before the <tbody> so the browser could render the footer before downloading potentially thousands of rows. In HTML5, it should be placed at the end, after the body.
HTML semantic_table.html 
<table class="artifact-table">
    <caption>Language Performance Metrics</caption>
    
    <thead>
        <tr>
            <th scope="col">Artifact ID</th>
            <th scope="col">Language</th>
            <th scope="col">Entropy Score</th>
        </tr>
    </thead>
    
    <tbody>
        <tr>
            <th scope="row">ARV-01</th>
            <td>Rust</td>
            <td>0.98</td>
        </tr>
        <tr>
            <th scope="row">ARV-02</th>
            <td>C++</td>
            <td>0.94</td>
        </tr>
    </tbody>
    
    <tfoot>
        <tr>
            <td colspan="2">System Average</td>
            <td>0.96</td>
        </tr>
    </tfoot>
</table>

Header Scope & Accessibility

Simply using the <th> (Table Header) tag is not enough for high-end architecture. We must use the scope attribute to tell the browser's accessibility tree whether the header applies to the column or the row.

  • scope="col": The header for the vertical column.
  • scope="row": The header for the horizontal row (often used for the first cell of each row).

The Caption Element

The <caption> element provides a title for the table. It must be the first child of the <table>. Like the <figcaption> for images, it links a human-readable description to the data structure.

⚡
Architect's Design Tip: Tables are notoriously difficult to make responsive. In the ROSETTA project, we recommend wrapping the <table> in a <div style="overflow-x: auto"> to ensure that on mobile devices, the user can swipe the data artifact horizontally without breaking the page layout.
