4.2 Citations & Edits

The Audit Trail: Marking provenance, modifications, and intellectual debt.

The Architecture of Quotation

Quoting external content is a fundamental part of the web. HTML provides two distinct mechanisms for this, depending on the volume of the text.

  • Blockquote: For lengthy, block-level quotations that break the flow of the text. By default, browsers indent this content.
  • Q (Quote): For short, inline quotations. Modern browsers automatically surround this element with quotation marks (" ") based on the document's language.
HTML quotation_logic.html 
<!-- Block Level -->
<blockquote cite="https://w3.org/spec">
    <p>The HTML specification is a living standard.</p>
</blockquote>

<!-- Inline Level -->
<p>
    As the architect said, <q>Semantics is the foundation.</q>
</p>

The <cite> Misconception

The <cite> element is one of the most misused tags in HTML.

Correct Usage: It represents the Title of a Work (e.g., a book title, a movie name, a research paper title).

Incorrect Usage: It is not used for the author's name.

HTML citation_standard.html 
<!-- ✅ CORRECT -->
<p>More info in <cite>The Art of Computer Programming</cite> by Knuth.</p>

<!-- ❌ WRONG -->
<p>Written by <cite>Donald Knuth</cite></p>

Tracking Revisions: <ins> and <del>

HTML has built-in support for "Track Changes" or diffs. This is incredibly useful for legal documents, collaborative editing apps, or changelogs.

  • <del>: Content that has been removed. Usually rendered with a strikethrough.
  • <ins>: Content that has been added. Usually rendered with an underline.

Both elements support the datetime attribute to record exactly when the edit happened, and the cite attribute to link to an explanation of the edit.

HTML version_control.html 
<p>
    Project ROSETTA will launch in 
    <del datetime="2025-01-01">2025</del> 
    <ins datetime="2026-02-15">2026</ins>.
</p>
⚡
Architect's Tip: Do not use <del> just to create a strikethrough visual effect (like a "discounted price"). For pure visuals, use the <s> tag. Use <del> only when the data is actually retracted or obsolete.
