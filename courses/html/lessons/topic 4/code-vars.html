4.3 Code, Preformatting & Variables

The Syntax of Syntax: Semantics for displaying technical data and computer interaction.

The Fragment: <code>

The <code> element represents a fragment of computer code. By default, browsers render this in a monospace font (like Courier or Consolas).

However, <code> is an inline element. It does not preserve whitespace. If you put multiple lines of code inside it, the browser will collapse them into a single line.

The Block: <pre> + <code>

To display a block of code while preserving indentation, line breaks, and tabs, we must combine two elements.

  1. <pre> (Preformatted Text): Preserves whitespace and prevents text wrapping.
  2. <code>: Semantically defines the content as code.

Standard practice for syntax highlighters (like Prism.js or Highlight.js) is to add a class to the code tag, such as class="language-css".

HTML block_syntax.html 
<!-- The Industry Standard Pattern -->
<pre>
    <code class="language-javascript">
function init() {
    console.log("Hello World");
}
    </code>
</pre>

User Input: <kbd>

The <kbd> element represents user input, typically keyboard entry or voice commands.

It is often styled to look like a physical key. Architects frequently nest <kbd> tags to represent key combinations.

HTML shortcuts.html 
<p>
    To save the file, press 
    <kbd><kbd>Ctrl</kbd> + <kbd>S</kbd></kbd>
</p>

System Output & Variables

Two lesser-known but semantically vital tags:

  • <samp> (Sample Output): Represents output from a computer program (e.g., "File not found" or terminal logs).
  • <var> (Variable): Represents a mathematical variable (x, y) or a programming variable placeholder. Usually rendered in italics.
HTML terminal_simulation.html 
<p>
    The computer calculated that <var>x</var> = 5.
    Then it printed: <samp>Process Complete.</samp>
</p>
