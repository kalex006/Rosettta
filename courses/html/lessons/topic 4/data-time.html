4.4 Time, Data & Address

Temporal Semantics: Translating human ambiguity into machine precision.

The Temporal Anchor: <time>

Humans write dates loosely ("Jan 21st", "Yesterday", "Two weeks ago"). Machines cannot index relative time efficiently.

The <time> element resolves this by accepting a machine-readable datetime attribute (following the ISO-8601 standard) while displaying a human-readable string to the user.

HTML iso_standards.html 
<!-- Specific Date -->
<p>
    The project launched on 
    <time datetime="2026-01-15">January 15th</time>.
</p>

<!-- Date and Time (T Separator + Timezone Offset) -->
<p>
    Meeting starts at 
    <time datetime="2026-05-20T14:30:00+05:30">2:30 PM (IST)</time>.
</p>

<!-- Duration (P = Period, 2H = 2 Hours) -->
<p>
    The exam lasts 
    <time datetime="PT2H30M">2 hours and 30 minutes</time>.
</p>

The Value Binder: <data>

The <data> element is the generic version of <time>. It binds a machine-readable value to a piece of content.

This is extremely useful for E-commerce (Product SKUs) or Library Systems (ISBNs), where the user sees a name, but the script needs an ID number.

HTML inventory_logic.html 
<ul>
    <li>
        <data value="SKU-998877">Dark Mode Mechanical Keyboard</data>
    </li>
    <li>
        <data value="ISBN-9780321751041">The Art of Computer Programming</data>
    </li>
</ul>

The Contact Context: <address>

We touched on this in Module 3, but it bears repeating in the context of text semantics. The <address> tag is not for arbitrary postal addresses (like a shipping address form).

It specifically defines the Contact Information for the Author/Owner of the surrounding <article> or <body>.

By default, browsers often render this in italics, implying it is "meta-data" about the document itself.
